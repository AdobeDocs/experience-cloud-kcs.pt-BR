# Como otimizar os índices do Oak Lucene para reduzir o tempo de indexação

## Descrição


Otimize os índices de lucene do Oak AEM para reduzir o tempo de reindexação.
<br><br>Ambiente<br><br>
AEM 6.0 + Oak 1.0.16 ou posterior AEM 6.1 e AEM 6.2


## Resolução


Você pode otimizar os índices Oak Lucene ajustando o oak: QueryIndexDefinition com propriedades como &quot;includedPaths&quot;, &quot;excludedPaths&quot;, &quot;name&quot; e &quot;indexPath&quot;.

- As propriedades &quot;includedPaths&quot; e &quot;excludedPaths&quot; restringem quais caminhos o índice atravessa ao reindexar e a quais caminhos ele escuta para atualizações de índice.
- As propriedades &quot;name&quot; e &quot;indexPath&quot; facilitam a compreensão dos logs de índice, pois vinculariam as mensagens de log ao caminho ou nome do índice.  Por exemplo:



| `30.01.2017 14:07:12.934 *DEBUG* pool-8-thread-2 org.apache.jackrabbit.oak.plugins.index.lucene.LuceneIndexEditor damassetlucene(` `/oak` `:index` `/damAssetLucene` `) = Indexed 9000 nodes...` |
| --- |


<b>Etapas para otimizar os índices</b>:

1. Vá para http://*aem-host:port*/crx/de/index.jsp e faça logon como administrador
2. Navegue até /oak:index
3. Adicione as propriedades realçadas abaixo 1 aos índices correspondentes.  O <b>includedPaths</b> e <b>excludedPaths</b> As propriedades devem ser string com vários valores ( String )
4. Clique em &quot;[!UICONTROL Salvar tudo]&quot; no CRXDe
