# Como configurar o Adobe I/O: Autenticação - passo a passo

## Descrição

<br>Este documento mostra como criar certificados, chaves e tokens para poder executar.<br><br><br><br><br>Objetivo<br><br><br><br> <br><br>
Para fornecer etapas claras e reproduzíveis para gerar um token de portador do Adobe I/O para executar chamadas de API para soluções de DMA, como [!DNL Target].


## Resolução

<br><br>Etapas<br><br><br><br> <br><br>
<b>Etapas para criar um token de portador para executar chamadas de API do Adobe I/O:</b>

1. Gerar chave privada e certificado público
2. Criar integração no console do Adobe I/O
3. Gerar token JWT
4. Trocar token JWT por um token de portador de acesso
5. Token de portador de acesso do usuário para executar chamadas de API


- <b>Gere a chave privada e o certificado público de acordo com a seguinte documentação:</b> https://www.adobe.io/apis/cloudplatform/console/authentication/createcert.html


   Em um Mac, os seguintes comandos são criados no terminal. 

   Em um computador, você precisa baixar o [!DNL Cygwin] (ou outra ferramenta de preferência pessoal) e executá-lo a partir da linha de comando. Estas são as etapas para baixar e instalar o [!DNL Cygwin]:

   1. Navegue até https://cygwin.com/install.html
   2. Baixe e execute o `setup-x86_64.exe`

   <b>*Observação</b>:  Seu diretório inicial é: `C:\cygwin64\home\WINDOWSUSER`

   Você pode procurar e instalar pacotes adicionais durante a fase de instalação. Recomendo instalar tudo relacionado a “`curl`” e “`ssh.`”*


   Este é o comando para executar:
   <br> <br> <br>

   | `$ openssl req -nodes -text -x509 -newkey rsa:2048 -keyout secret.pem -out certificate.pem -days 356` |
   | --- |

<br> 

    &lt;b>Observação&lt;/b>:  ele faz várias perguntas que você precisa preencher para gerar o certificado, veja a captura de tela abaixo:
![rtaImage](https://helpx.adobe.com/content/dam/help/en/marketing-cloud-core/kb/adobe-io-authentication-step-by-step/jcr%3acontent/main-pars/procedure/proc_par/step_0/step_par/image/rtaImage.png "rtaImage")<br> 

    Após a criação dos arquivos, converta a “secret.key” em “secret.pem” usando o seguinte comando:
<br> <br> <br>

    | `$ openssl pkcs8 -topk8 -inform PEM -outform DER -` `in` `secret.pem  -nocrypt  secret.key` |
    | --- |

<br> 

    Estes são os arquivos que são criados em seu sistema de arquivos (você pode mover esses arquivos para outro local em sua máquina neste ponto por organização):
