# Descubra quais solicitações AEM o cache do Dispatcher

## Descrição

Quais solicitações e respostas HTTP fazem AEM cache do Dispatcher? (AEM Dispatcher 4.3.3)<br><br><br>

## Resolução


A documentação abrange os cenários mais importantes que afetam a capacidade de cache de solicitações e respostas que passam pelo módulo Dispatcher AEM. Todas as regras de cache não são abordadas.
<br><br><br><br> <br><br>Solicitação HTTP e regras de cache:<br><br><br><br> <br><br>
Para que uma solicitação HTTP possa ser armazenada em cache pelo dispatcher, ela deve seguir as seguintes regras.
<br><br><br><br> <br><br>1. O URL É ABSOLUTO (NÃO COMEÇA COM . OU ..), TEM UMA EXTENSÃO DE ARQUIVO E O NOME DO ARQUIVO NO URL FUNCIONA COMO UM ARQUIVO NO SISTEMA DE ARQUIVOS.<br><br><br><br> <br><br>
<b>Mensagens de log relacionadas:</b>
<br><br><br><br><br> <br><br> <br><br><br><br>

| `URI not canonical: %s.``Unable to map URI to ``file``: %s.``cache ``file` `path too long``temporary ``file` `path too long``request URL has no extension` |
| --- |

<br><br><br><br><br> <br><br>
<b>Exemplos:</b>
<br><br><br><br> <br><br>

| /content/test | não armazenado em cache |
| --- | --- |
| /content/test.html | em cache |

<br><br><br><br> <br><br>2. NÃO TEM ESCRAVOS APÓS A PRIMEIRA EXTENSÃO DE ARQUIVO.<br><br><br><br> <br><br>
<b>Mensagens de log relacionadas:</b>
<br><br><br><br><br> <br><br> <br><br><br><br>

| `request URL has trailing slash` |
| --- |

<br><br><br><br><br> <br><br>
<b>Exemplos:</b>
<br><br><br><br> <br><br>

| /content/test.html/ | não armazenado em cache |
| --- | --- |
| /content/test.html/foo.jpg | não armazenado em cache |
| /content/test.html | em cache |

<br><br><br><br> <br><br>3. ELE USA O MÉTODO HTTP GET OU HEAD.<br><br><br><br> <br><br>
<b>Mensagens de log relacionadas:</b>
<br><br><br><br><br> <br><br> <br><br><br><br>

| `request method is neither GET nor HEAD` |
| --- |

<br><br><br><br><br> <br><br>
<b>Exemplos:</b>
<br><br><br><br> <br><br>

| HEAD /content/test.html HTTP/1.1 | em cache |
| --- | --- |
| GET /content/dam/test.jpg HTTP/1.1 | não armazenado em cache |
| POST /content/test.html HTTP/1.1 | não armazenado em cache |

<br><br><br><br> <br><br>4. A SOLICITAÇÃO É NEGADA POR /CACHE = /REGRAS DEFINIDAS NA CONFIGURAÇÃO DO DISPATCHER .ANY FARM.<br><br><br><br> <br><br>
<b>Mensagens de log relacionadas:</b>
<br><br><br><br><br> <br><br> <br><br><br><br>

| `URI not ``in` `cache rules: %s``request URL not ``in` `cache rules` |
| --- |

<br><br><br><br><br> <br><br>
<b>Exemplos:</b>
<br><br><br><br> <br><br>

| O farm tem /cache = /rules e uma solicitação é recebida para <b>/content/test.html</b><br><br>/cache<br><br>{<br><br>...<br><br>/rules<br><br>{<br><br>/0001 {<br><br>tipo &quot;negar&quot;<br><br>glob &quot;\*&quot;<br><br>}<br><br>} | não armazenado em cache |
| --- | --- |
| O farm tem /cache = /rules e uma solicitação é recebida para <b>/content/test.html</b><br><br>/cache<br><br>{<br><br>...<br><br>/rules<br><br>{<br><br>/0001 {<br><br>tipo &quot;allow&quot;<br><br>glob &quot;\*&quot;<br><br>}<br><br>} | em cache |

<br><br><br><br> <br><br>5. A SOLICITAÇÃO HTTP NÃO CONTÉM UM CABEÇALHO DE AUTORIZAÇÃO OU CONTÉM O CABEÇALHO, MAS /ALLOWAUTHORIZED É DEFINIDO COMO 1 NA CONFIGURAÇÃO DO FARM.<br><br><br><br> <br><br>
<b>Mensagens de log relacionadas:</b>
<br><br><br><br><br> <br><br> <br><br><br><br>

| `request contains authorization` |
| --- |

<br><br><br><br><br> <br><br>
<b>Exemplos:</b>
<br><br><br><br> <br><br>

| Solicitação HTTP:<br><br>GET /content/test.html HTTP/1.1<br>  Autorização: YWxhZGRpbjpvcGVuc2VzYW1l básico<br><br>dispatcher.any farm /cache = /allowAuthorized configuration:<br><br>/cache {<br>    /allowAuthorized &quot;0&quot;<br>  } | não armazenado em cache |
| --- | --- |
| Solicitação HTTP:<br><br>GET /content/test.html HTTP/1.1<br>  Autorização: YWxhZGRpbjpvcGVuc2VzYW1l básico<br><br>dispatcher.any farm /cache = /allowAuthorized configuration:<br><br>/cache {<br>    /allowAuthorized &quot;1&quot;<br>  } | em cache |

<br><br><br><br> <br><br>6. A SOLICITAÇÃO HTTP CONTÉM TOKEN DE LOGON OU COOKIE DE AUTORIZAÇÃO NO CABEÇALHO DO COOKIE OU CONTÉM (OU AMBOS) DESSES COOKIES E /ALLOWAUTHORIZED ESTÁ DEFINIDO COMO 1 NA CONFIGURAÇÃO DO FARM.<br><br><br><br> <br><br>
<b>Mensagens de log relacionadas:</b>
<br><br><br><br><br> <br><br> <br><br><br><br>

| `request contains authorization` |
| --- |

<br><br><br><br><br> <br><br>
<b>Exemplos:</b>
<br><br><br><br> <br><br>
<b>Mensagem de log relacionada:</b>
<br><br><br><br> <br><br>
<b>Exemplos:</b>
<br><br><br><br> <br><br>

| Solicitação HTTP:<br><br>GET /content/test.html HTTP/1.1<br>  Cookie: login-token=...<br><br>dispatcher.any farm /cache = /allowAuthorized configuration:<br><br>/cache {<br>    /allowAuthorized &quot;0&quot;<br>  } | não armazenado em cache |
| --- | --- |
| Solicitação HTTP:<br><br>GET /content/test.html HTTP/1.1<br>  Cookie: authorization=YWxhZGRpbjpvcGVuc2VzYW1l<br><br>dispatcher.any farm /cache = /allowAuthorized configuration:<br><br>/cache {<br>    /allowAuthorized &quot;0&quot;<br>  } | não armazenado em cache |
| Solicitação HTTP:<br><br>GET /content/test.html HTTP/1.1<br>  Cookie: login-token=...<br><br>dispatcher.any farm /cache = /allowAuthorized configuration:<br><br>/cache {<br>    /allowAuthorized &quot;1&quot;<br>  } | em cache |

<br><br><br><br> <br><br>7. HÁ UM PARÂMETRO DE SEQUÊNCIA DE CONSULTA NO URL E O PARÂMETRO PODE SER IGNORADO POR MEIO DA CONFIGURAÇÃO /IGNOREURLPARAMS DO FARM.<br><br><br><br> <br><br>
<b>Mensagens de log relacionadas:</b>
<br><br><br><br><br> <br><br> <br><br><br><br>

| `request contains a query string` |
| --- |

<br><br><br><br><br> <br><br>
<b>Exemplos:</b>
<br><br><br><br> <br><br>

| Solicitação HTTP:<br><br>GET /content/test.html?test=1 HTTP/1.1<br><br>Configuração de farm/cache:<br><br>/ignoreUrlParams {<br>     /0001 { /type &quot;allow&quot; /glob &quot;\*&quot; } <br>  } | em cache |
| --- | --- |
| Solicitação HTTP:<br><br>GET /content/test.html?test=1 HTTP/1.1<br><br>Configuração de farm/cache:<br><br>/ignoreUrlParams {<br>     /0001 { /type &quot;deny&quot; /glob &quot;\*&quot; } <br>     /0001 { /type &quot;allow&quot; /glob &quot;test&quot; }<br>  } | em cache |
| Solicitação HTTP:<br><br>GET /content/test.html?test=1 HTTP/1.1<br><br>dispatcher.any farm /cache = /allowAuthorized configuration:<br><br>/ignoreUrlParams {<br>     /0001 { /type &quot;deny&quot; /glob &quot;\*&quot; } <br>     /0001 { /type &quot;allow&quot; /glob &quot;q&quot; }<br>  } | não armazenado em cache |

<br><br><br><br> <br><br>Resposta HTTP e regras de cache:<br><br><br><br> <br><br>
A resposta HTTP retornada do AEM pode ser armazenada em cache se os seguintes critérios forem atendidos.
<br><br><br><br> <br><br>1. O DISPATCHER PODE ENVIAR E RECEBER UMA RESPOSTA 200 OK A PARTIR DE UMA DAS &quot;RENDERIZAÇÕES&quot; DEFINIDAS.<br><br><br><br><br><br>
Observação:

Se não [/timeout](https://helpx.adobe.com/experience-manager/dispatcher/using/dispatcher-configuration.html#DefiningPageRenderersrenders) está definido em /renders ou está definido como <b>0</b> então, ela esperaria para sempre por uma conexão com a instância de AEM mesmo se a instância estivesse inativa.


<br><br><br><br> <br><br>
<b>Mensagens de log relacionadas:</b>
<br><br><br><br><br> <br><br> <br><br><br><br>

| `Unable to send request to remote server.``Unable to receive response from remote server.``Remote server returned: %s``No backend available.` |
| --- |

<br><br><br><br><br> <br><br>
<b>Exemplos:</b>
<br><br><br><br> <br><br>

| Solicitação HTTP:<br><br>200 OK | em cache |
| --- | --- |
| Solicitação HTTP:<br><br>404 Página não encontrada | não armazenado em cache |

<br><br><br><br> <br><br>2. NENHUM DESSES CABEÇALHOS DE RESPOSTA ESTÁ PRESENTE NA RESPOSTA:<br><br><br><br> <br><br>
- <b>Dispatcher: no-cache</b>
- <b>Controle de cache: no-cache</b>
- <b>Pragma: no-cache</b>

<br><br><br><br> <br><br>
<b>Mensagem de log relacionada:</b>
<br><br><br><br><br> <br><br> <br><br><br><br>

| `Backend forbids caching: %s, sent: %s"``response contains no_cache` |
| --- |

<br><br><br><br><br> <br><br>
<b>Exemplos:</b>
<br><br><br><br> <br><br>

| Resposta HTTP:<br><br>200 OK<br>  Dispatcher: no-cache | não armazenado em cache |
| --- | --- |
| Resposta HTTP:<br><br>200 OK<br>  Controle de cache: no-cache | não armazenado em cache |
| Resposta HTTP:<br><br>200 OK<br>  Pragma: no-cache | não armazenado em cache |
| Resposta HTTP:<br><br>200 OK | em cache |

<br><br><br><br> <br><br>3. O VALOR DE COMPRIMENTO DE CONTEÚDO É MAIOR QUE ZERO BYTES.<br><br><br><br> <br><br>
<b>Mensagem de log relacionada:</b>
<br><br><br><br><br> <br><br> <br><br><br><br>

| `response content length is zero` |
| --- |

<br><br><br><br><br> <br><br>
<b>Exemplos:</b>
<br><br><br><br> <br><br>

| Resposta HTTP:<br><br>200 OK<br>  Duração do conteúdo: 0 | não armazenado em cache |
| --- | --- |
| Resposta HTTP:<br><br>200 OK<br>  Duração do conteúdo: 3120 | em cache |
