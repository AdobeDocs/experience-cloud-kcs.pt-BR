# Audience Manager to [!DNL Campaign]

## Descrição


Este guia de implementação técnica fornece informações sobre a arquitetura de referência, diretrizes, etapas de configuração e testes que os especialistas de implementação precisam seguir ao integrar o Audience Manager com o [!DNL Campaign].



Este guia de integração técnica destina-se a consultores de Adobe, parceiros de Adobe e outros especialistas em implementação experientes. A configuração dessa integração requer um entendimento completo do processo de implementação para cada solução individual abordada neste guia.

Os clientes do Adobe podem entrar em contato com [Adobe Professional Services](http://www.adobe.com/experience-cloud/service-support/professional-services/contact-us.html) ou [Parceiro Adobe](http://www.adobe.com/experience-cloud/service-support/partners.html) para saber mais sobre essa integração.




## Resolução


<b>Visão geral da integração</b>

A integração usa públicos compartilhados do Marketing Cloud e a funcionalidade de ID declarada no AAM.

<b>Planejamento De Solução E Arquitetura De Integração</b>

A. Pré-requisitos (Licença, Conectividade, Inicialização)

1. Organização do Marketing Cloud provisionada para o cliente
2. Licença Adobe Analytics
3. Adobe Campaign V6 a ser atualizado para 8687 ou superior (8705 ou superior recomendado)
4. Licença Adobe Audience Manager
5. Adobe DTM
6. Públicos-alvo compartilhados da Adobe Marketing Cloud
7. Dados do CRM do cliente a serem enviados para o Adobe Campaign
8. O Adobe Campaign v6 deve ser conectado à Marketing Cloud
9. Configurar [!DNL Campaign] usuários no Marketing Cloud
10. Os usuários devem poder fazer logon no [!DNL Campaign] V6 via Adobe ID


B. Arquitetura de implementação

![1](https://helpx.adobe.com/content/dam/help/en/marketing-cloud/how-to/aam-campaign/_jcr_content/main-pars/image/1.png "1")

<b>Instalação, provisionamento e configuração</b>

A. Configuração/provisionamento

1. Qualquer provisionamento de serviços principais de soluções adicionais deve ser feito por meio do representante do Adobe.
2. [!DNL Campaign] O provisionamento precisa ser feito por [!DNL Campaign] TechOps.
3. Fontes de dados, destinos AAM interface do usuário e AAM sFTP devem ser criados/provisionados por meio do representante do Adobe.


B. Implementação de soluções

1. CRM do cliente

O cliente precisa ter um sistema que armazene todos os dados do CRM. Essa deve ser uma lista de endereços de email, combinada com uma ID com hash exclusiva desse cliente. Por exemplo: *[abc@abcmail.com](mailto:abc@abcmail.com), 1231231j2g3123<b>[xyz@xyzmail.com](mailto:xyz@xyzmail.com), 234234jh23k4jh324* Onde a primeira coluna é o endereço de email e a segunda coluna é o CRM_ID com hash

2. Sincronizar o banco de dados do CRM com [!DNL Campaign] O cliente precisa configurar um processo no qual envie seus dados de CRM para [!DNL Campaign]. O email, CRM_ID, são os campos mínimos a serem sincronizados.

3. Disponibilize o CRM_ID na página. Isso requer que o cliente tenha feito logon. Após o logon, o CMS deve entrar em contato com o CRM do cliente para verificar se a senha está correta. Nesse nível, também o CRM_ID exclusivo deve ser capturado e enviado para a página, de preferência para um data mail.

4. O DTM lê a Configuração do CRM_ID na ferramenta &quot;Serviço de ID do Marketing Cloud&quot;. O elemento de dados hash_email é configurado no DTM lendo o CRM_ID disponível no dataLayer.

![2](https://helpx.adobe.com/content/dam/help/en/marketing-cloud/how-to/aam-campaign/_jcr_content/main-pars/procedure/proc_par/step_253280329/step_par/image/2.jpg "2")

5. Configurar AAM sFTP no Adobe Campaign

O sFTP criado para o AAM precisa ser configurado com o Adobe Campaign para que [!DNL Campaign] O pode enviar e extrair segmentos do AAM. As credenciais sFTP serão fornecidas a você pelo Atendimento ao cliente do Adobe ou pelo representante do Adobe. Durante o provisionamento, os locais de entrada e saída do ftp são provisionados e são usados por [!DNL Campaign] para enviar públicos-alvo para AAM e segmentos integrados do AAM para [!DNL Campaign] respectivamente.

![3](https://helpx.adobe.com/content/dam/help/en/marketing-cloud/how-to/aam-campaign/_jcr_content/main-pars/procedure/proc_par/step_801238358/step_par/image/3.jpg "3")

6. Criar as fontes de dados e os destinos com o AAM

As Fontes de dados e os destinos das integrações devem ser criados na interface do usuário do AAM, se ainda não tiverem sido criados. Se isso tiver sido criado, as IDs de origem de dados e de destino deverão ser usadas para configurá-las na interface do usuário do Adobe Campaign. Duas fontes de dados devem ser criadas, uma com base na ID do Marketing Cloud e outra com base na ID declarada. Os destinos são criados na interface do administrador do AAM correspondente a cada fonte de dados. Isso exigiria o envolvimento de um consultor de AAM ou AAM Clientcare.

![4](https://helpx.adobe.com/content/dam/help/en/marketing-cloud/how-to/aam-campaign/_jcr_content/main-pars/procedure/proc_par/step_1253393284/step_par/image/4.jpg "4")

7. Configurar as fontes de dados no Adobe Campaign As fontes de dados são configuradas no Adobe Campaign. Duas fontes de dados serão criadas na interface do usuário do Adobe Campaign; um com base na ID declarada ou na ID do cliente com hash e o outro com base na ID do Marketing Cloud ou na UUID do solicitante. A ID de destino correspondente a cada ID é adicionada nos respectivos campos. Por exemplo, a ID de destino AAM Adobe Campaign de destino - ID declarada é adicionada na configuração da fonte de dados com a chave de reconciliação como ID do cliente com hash.

- Não tenho a opção &quot;criar público-alvo compartilhado&quot; em listas, qual pode ser o motivo?


Você pode não ter feito logon usando sua Adobe ID para [!DNL Campaign]. Se você fez logon usando sua Adobe ID, certifique-se de ter sido adicionado ao produto da Biblioteca de público-alvo na interface do usuário do Marketing Cloud Enterprise Console.

- Compartilhei um segmento com o AAM, mas ainda não o vejo na interface do usuário AAM. Qual poderia ser a razão?


Depois que o fluxo de trabalho é executado, pode levar de 12 a 24 horas para que os dados sejam assimilados no AAM e de 24 a 48 horas adicionais para que os relatórios sejam atualizados na interface do usuário do AAM. Se você não vir um número sob as características/segmentos AAM interface do usuário, mesmo depois de 3 dias, entre em contato com seu consultor de AAM. </b>                                   <b>



</b>Práticas recomendadas<b>

1. Recomenda-se que o AAM e [!DNL Campaign] os consultores trabalham juntos para concluir a configuração em ambas as extremidades.
2. É recomendável usar um segmento com registros limitados para testar a integração de ponta a ponta.
3. Nunca mapeie os segmentos para a [!DNL Campaign] destino da interface do usuário do AAM
