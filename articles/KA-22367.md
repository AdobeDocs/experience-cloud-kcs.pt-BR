# Erro SAML - o parâmetro de resposta SAML não foi fornecido ou é inválido

## Descrição {#description}

After verifying the customer&#39;s SAML configuration it was noticed that whenever they were trying to login to AEM author, the 403 error was received.

O erro 403 geralmente aparece se o Filtro referenciador do Apache Sling não estiver ativado. Verificando a configuração do Filtro referenciador Apache Sling a opção &quot;Permitir vazio&quot; não foi marcada e também não havia host adicionado em &quot;Permitir hosts&quot;

Para que o logon SAML funcione, o nome de host do Provedor de identidade precisa ser adicionado à configuração do OSGi do Filtro do referenciador do Apache Sling.

Após concluir estas etapas necessárias - marque a opção &quot;Permitir vazio&quot; e adicione os hosts que representam o provedor de IdP à opção &quot;Permitir hosts&quot;. O logon foi bem-sucedido.



OBSERVAÇÃO:

O URL do IDP deve ser adicionado à configuração do Filtro referenciador do Apache Sling sem incluir o protocolo, por exemplo, aem-sso-saml em vez de https://aem-sso-saml
